AWS Messaging

SNS + SQS
Pub/Sub Pattern

Publisher ส่งครั้งเดียว — SNS กระจายไปทุก Queue โดยอัตโนมัติ Consumer แต่ละตัวทำงานอิสระ

SNS — Publish & Fan-out
SQS — Buffer & Store
Consumer — Pull & Process
ขั้นตอนการทำงาน
01 — Publish
ส่งออเดอร์เข้า SNS
publish_order.py สร้างข้อมูล Order + UUID แล้วยิง HTTPS ไปที่ SNS Topic ผ่าน ARN
02 — Fan-out
SNS กระจายข้อความ
SNS ส่งสำเนาไปยังทุก Queue ที่ Subscribe ไว้พร้อมกัน ทั้ง Inventory และ Email
03 — Buffer
SQS เก็บรอ Consumer
Queue เก็บข้อมูลได้นาน 4 วัน (default) ข้อความไม่หายแม้ Consumer จะออฟไลน์
04 — Process
Consumer ดึงและประมวลผล
process_inventory.py วน Long Poll ดึงข้อมูล ตัดสต็อก แล้วลบออกจาก Queue

publish_order.py
import boto3, json, uuid, config

def publish_order():
  sns = boto3.client(
    'sns',
    aws_access_key_id=config.AWS_ACCESS_KEY_ID,
    aws_secret_access_key=config.AWS_SECRET_ACCESS_KEY,
    region_name=config.AWS_REGION,
  )
  order = {
    "order_id": str(uuid.uuid4()),
    "item": "Mechanical Keyboard",
    "quantity": 1,
  }
  sns.publish(
    TopicArn=config.SNS_TOPIC_ARN,  # ARN
    Message=json.dumps(order),
  )
